<h2>궁금한 점</h2>

<h3>1. css에서 이미지를 사이즈에 맞게 늘리고 싶은데 object-fit이랑 background-size 중에 뭐 써야해?</h3>

> `<img>` 태그를 써서 이미지를 넣었으면 object-fit  
> css에서 이미지를 넣었으면 background-size

| 비교 항목  | object-fit                                   | background-size                         |
| ---------- | -------------------------------------------- | --------------------------------------- |
| 사용 대상  | `<img>, <video>` 태그                        | 모든 HTML 요소의 배경                   |
| 용도       | 콘텐츠로서의 이미지 (제품 사진, 프로필 등)   | 디자인적인 배경 (패턴, 장식 등)         |
| SEO/접근성 | 이미지에 alt 속성을 줄 수 있어 검색에 유리함 | 검색 엔진이 인식하지 못함               |
| 성능       | 브라우저 렌더링이 약간 더 빠름               | 여러 이미지를 겹치거나 조절할 때 유연함 |

<h3>2. .center와 .slider가 동일한 요소를 가리키는데, 굳이 따로 쓰는 이유는 무엇일까</h3>

**(1) 역할 분담 (관심사 분리)**

두 클래스는 해당 요소에서 완전히 다른 역할을 담당하고 있습니다.

| 클래스이름                  | 역할                                                                                      |
| --------------------------- | ----------------------------------------------------------------------------------------- |
| .center (배치 담당)         | 이 요소가 화면의 어디에 위치할지를 결정합니다. (중앙 정렬)                                |
| .slider (외형 및 기능 담당) | 이 요소가 어떻게 생겼고(배경, 크기, 그림자), 어떤 동작(애니메이션)을 하는지를 결정합니다. |

**(2) 코드의 재사용성**

만약 .center라는 클래스를 따로 만들어두면, 슬라이더뿐만 아니라 다른 요소도 화면 중앙에 보내고 싶을 때 아주 간편하게 재사용할 수 있습니다.

**(3) 애니메이션 유지보수**

현재 코드에는 animation: animate 20s infinite;가 포함되어 있습니다. 만약 중앙 정렬을 위한 transform: translate(-50%, -50%)와 애니메이션에서 사용하는 transform이 충돌하게 되면 애니메이션이 씹히거나 위치가 어긋나는 문제가 생길 수 있습니다.

이를 방지하기 위해 위치를 잡는 속성(.center)과 움직임을 주는 속성(.slider)을 논리적으로 구분해 두는 것이 안전합니다.
